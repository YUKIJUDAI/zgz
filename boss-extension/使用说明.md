## 简介

**Boss直聘求职助手**是一款专为求职者打造的Chrome浏览器扩展，可以帮助您：

- ✅ 自动筛选匹配的职位，节省浏览时间
- ✅ 智能生成个性化打招呼消息
- ✅ 批量向合适的职位发送沟通请求
- ✅ 实时接收面试邀请通知
- ✅ 精确计算通勤距离和时间

**适用人群**：正在Boss直聘上找工作的求职者，特别是技术岗位（前端、后端等）

## 安装步骤

### 方法一：开发者模式安装（推荐）

1. **打开Chrome扩展管理页面**
   - 在Chrome浏览器地址栏输入：`chrome://extensions/`
   - 或点击右上角三个点 → 更多工具 → 扩展程序

2. **开启开发者模式**
   - 找到页面右上角的"开发者模式"开关
   - 点击开启（变成蓝色）

3. **加载扩展**
   - 点击左上角"加载已解压的扩展程序"按钮
   - 选择 `boss-extension` 文件夹
   - 点击"选择文件夹"

4. **确认安装成功**
   - 在扩展列表中看到"Boss直聘求职助手"
   - 浏览器右上角出现扩展图标（紫色B）

### 方法二：打包安装

```bash
# 如果您熟悉命令行，可以先打包
zip -r boss-extension.zip boss-extension -x "*.git*"
```

然后在 `chrome://extensions/` 页面直接拖拽 zip 文件安装。

## 首次配置

### 1. 打开配置界面

点击浏览器右上角的扩展图标（紫色B） → 进入弹窗 → 点击"配置"标签页

### 2. 基本设置

#### 期望薪资
- 输入您期望的**最低月薪**（单位：元）
- 例如：`25000` 表示期望薪资至少25K
- 低于此薪资的职位会被降低评分

#### 必备技能
- 输入您掌握的**核心技能**，用逗号分隔
- 例如：`Vue,TypeScript,JavaScript,前端工程化`
- 职位描述中包含这些技能会获得高分

#### 加分技能
- 输入您了解的**加分技能**，用逗号分隔
- 例如：`React,Node.js,Webpack,Vite,小程序`
- 包含这些技能会额外加分

#### 排除关键词
- 输入您**不想看到**的关键词，用逗号分隔
- 例如：`外包,驻场,996,大小周,单休`
- 包含这些词的职位会被直接过滤（一票否决）

#### 匹配阈值
- 设置职位匹配的**最低分数**（60-100分）
- 例如：`60` 表示只推荐60分以上的职位
- 建议设置：60-70分（太高可能错过机会）

### 3. 打招呼设置

#### 启用自动打招呼
- 勾选此选项以开启自动打招呼功能
- 不勾选则只进行职位匹配，不会自动发送消息

#### 每日上限
- 设置每天最多打招呼的次数
- 建议：`20-30`次（避免被系统限制）
- 系统会自动记录，达到上限后停止

#### 自定义开场白（可选）
- 如果您想使用固定的开场白，可以在这里填写
- 留空则使用智能生成的个性化开场白
- 示例：
  ```
  您好！我是一名有5年经验的前端工程师，
  看到贵司的职位觉得很匹配。
  我熟悉Vue/React/TypeScript技术栈，
  期望薪资25-35K，目前随时可到岗。
  期待和您详细沟通！
  ```

### 4. 个人资料

填写您的真实信息，用于生成打招呼消息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 姓名 | 您的真实姓名 | 金超宇 |
| 工作年限 | 工作经验年数 | 9 |
| 技术栈 | 主要技术方向 | Vue3/TypeScript/JavaScript |
| 联系电话 | 手机号码 | 18557519506 |
| 期望薪资范围 | K为单位 | 25-35 |

### 5. 通知设置（可选）

如果您希望收到面试邀请的实时通知，可以配置以下渠道：

#### 企业微信Webhook
- 适用于团队协作场景
- 获取方式：企业微信 → 群设置 → 群机器人 → 添加机器人 → 复制Webhook地址

#### Server酱密钥
- 可以将通知推送到您的微信
- 获取方式：访问 https://sct.ftqq.com/ → 登录 → 获取SendKey
- 格式：`SCTxxxxxxxxxxx`

#### Bark密钥
- iOS推送通知服务
- 下载Bark App → 获取密钥

**注意**：通知设置完全可选，不配置也不影响核心功能使用。

### 6. 保存配置

- 填写完所有配置后，点击页面底部的"💾 保存配置"按钮
- 看到"配置已保存"提示即表示成功
- 如果想恢复默认设置，点击"🔄 恢复默认"

## 功能使用

### 一、浏览职位列表（自动匹配）

1. **访问Boss直聘职位列表页**
   - 打开 https://www.zhipin.com/web/geek/job
   - 或在Boss直聘首页搜索职位

2. **自动初步评分**
   - 扩展会自动扫描页面上的所有职位
   - 在每个职位卡片右上角显示"XX分"标签（最高25分）
   - 评分依据：薪资匹配（15分）+ 地点匹配（10分）

3. **查看评分结果**
   - 🟢 绿色标签（25分）：薪资和地点完全匹配
   - 🔵 蓝色标签（15-24分）：部分匹配
   - ⚪ 灰色标签（0-14分）：条件不符

4. **查看详细信息**
   - 鼠标悬停标签可看详细评分
   - 点击职位进入详情页可进行精确分析（包含技能匹配）

### 二、查看职位详情（精确分析）

1. **点击进入职位详情页**
   - 从列表页点击任意职位进入

2. **查看渐进式评分**
   - **第一阶段**（立即显示）：基础条件匹配
     - 显示列表页缓存的初步分（薪资+地点）
     - 提示"正在精确分析..."

   - **第二阶段**（1-2秒后）：精确匹配度
     - 基于完整职位描述（JD）重新评分
     - 显示全维度评分（最高100分）：
       - 技能匹配（50分）- 必备技能在JD中的匹配度
       - 加分技能（20分）- 加分项匹配情况
       - 薪资匹配（15分）
       - 地点匹配（10分）
       - 标题匹配（5分）

   - 如果分数有明显提升，显示提示："✨ 精确分析后提升了 X 分！"

3. **查看匹配度面板**
   - 页面右侧自动显示匹配度面板
   - 包含总分和各项得分明细
   - 💡 提示：基于完整JD分析

4. **缓存优化**
   - 重复访问同一职位使用缓存，无需重新评分
   - 缓存有效期7天

5. **一键打招呼（高匹配职位）**
   - 如果匹配度达标且启用了打招呼功能
   - 页面右下角会显示"🤖 一键打招呼"按钮
   - 点击即可自动发送个性化消息

### 三、批量自动打招呼

1. **在职位列表页开启自动模式**
   - 点击扩展图标打开弹窗
   - 点击"🤖 开始自动打招呼"按钮

2. **自动执行流程**
   - 扫描页面上所有职位
   - 筛选出匹配度达标的职位
   - 按匹配度从高到低排序
   - 依次向每个职位发送打招呼消息
   - 遵守每日上限和操作间隔

3. **监控执行过程**
   - 页面右上角会显示toast提示
   - 显示当前处理进度（如：3/10）
   - 完成后显示成功数量

4. **自动停止条件**
   - 达到每日上限
   - 所有匹配职位处理完毕
   - 发生错误（会在日志中记录）

### 四、查看统计数据

点击扩展图标 → 仪表盘标签页：

- **打招呼**：今天已发送的消息数量
- **处理职位**：今天已查看/评分的职位数
- **匹配职位**：累计匹配成功的职位数

### 五、查看日志

点击扩展图标 → 日志标签页：

- 查看所有操作记录
- 筛选日志级别（信息/警告/错误）
- 导出日志用于问题排查
- 清空历史日志

## 高级功能

### 1. AI智能打招呼（可选）

如果您希望使用AI生成更个性化的打招呼语，可以配置AI服务：

#### 配置OpenAI

打开扩展数据目录，编辑配置添加：

```javascript
{
  "aiProvider": "openai",
  "aiModel": "gpt-4o-mini",
  "openaiApiKey": "sk-xxxxxxxxxxxxxxx",
  "openaiBaseURL": "https://api.openai.com/v1"
}
```

#### 配置Claude

```javascript
{
  "aiProvider": "claude",
  "aiModel": "claude-3-haiku-20240307",
  "claudeApiKey": "sk-ant-xxxxxxxxxxxxxxx",
  "claudeBaseURL": "https://api.anthropic.com/v1"
}
```

#### 配置本地模型（Ollama）

```javascript
{
  "aiProvider": "local",
  "aiModel": "llama2",
  "localBaseURL": "http://localhost:11434"
}
```

**注意**：AI功能需要自行申请API密钥，会产生费用。

### 2. 精确距离评估（可选）

配置高德地图API来精确计算通勤时间：

1. **获取高德API密钥**
   - 访问 https://lbs.amap.com/
   - 注册账号 → 创建应用 → 获取Key

2. **配置到扩展**

```javascript
{
  "enableMapAPI": true,
  "gaodeApiKey": "your-key-here",
  "homeAddress": "杭州市拱墅区都市水乡水曲苑",
  "maxCommuteTime": 40,  // 最大通勤时间（分钟）
  "maxDistance": 8000     // 最大直线距离（米）
}
```

3. **效果**
   - 自动计算公交/地铁通勤时间
   - 显示详细路线距离
   - 超过限制自动过滤

### 3. 实时通知配置

#### 企业微信机器人

1. 企业微信群 → 右键 → 添加群机器人
2. 选择"Webhook机器人"
3. 复制Webhook地址到配置中

#### Server酱（微信推送）

1. 访问 https://sct.ftqq.com/
2. 用GitHub登录
3. 复制SendKey到配置中

#### Bark（iOS推送）

1. App Store下载"Bark"
2. 打开App获取密钥
3. 填入配置中

## 常见问题

### Q1：为什么没有显示匹配度标签？

**可能原因**：
- 扩展未启用：检查弹窗中的开关是否打开
- 页面未加载完成：刷新页面重试
- 职位数量为0：当前页面没有职位卡片

**解决方法**：
1. 确认扩展已启用（图标不是灰色）
2. 刷新页面或重新搜索职位
3. 查看扩展的日志标签页，检查是否有错误信息

### Q2：打招呼失败怎么办？

**可能原因**：
- 已达每日上限
- 该职位今日已处理过
- 找不到聊天按钮（页面结构变化）
- 网络问题

**解决方法**：
1. 查看扩展日志（日志标签页）
2. 确认今日打招呼次数未超限
3. 手动点击"立即沟通"测试
4. 等待几分钟后重试

### Q3：如何修改匹配规则？

点击扩展图标 → 配置标签页 → 修改以下设置：
- 必备技能（影响技能匹配分）
- 期望薪资（影响薪资匹配分）
- 匹配阈值（影响筛选标准）

修改后点击"保存配置"即可生效。

### Q4：可以在其他浏览器使用吗？

目前仅支持Chrome浏览器和基于Chromium的浏览器（如Edge、Brave）。

### Q5：会不会被Boss直聘检测到？

本扩展已内置防检测机制：
- 随机操作间隔（3-8秒）
- 模拟真人点击和滚动
- 限制每日操作次数

**建议**：
- 不要设置过短的操作间隔
- 每日上限控制在20-30次
- 适度使用，不要过度依赖自动化

### Q6：数据存储在哪里？

所有数据（配置、日志、统计）都存储在本地浏览器的Chrome Storage中，不会上传到任何服务器。

卸载扩展后数据会被清除。

### Q7：如何备份配置？

1. 点击扩展图标 → 日志标签页
2. 点击"导出日志"按钮
3. 配置数据包含在导出的JSON文件中

## 注意事项

### ⚠️ 使用限制

1. **每日打招呼上限**
   - 建议：20-30次/天
   - 超过限制可能被系统检测

2. **操作间隔**
   - 默认：3-8秒随机间隔
   - 不建议修改为更短时间

3. **使用频率**
   - 不建议24小时持续使用
   - 工作时间使用效果更好

### ⚠️ 隐私安全

1. **数据存储**
   - 仅存储在本地浏览器
   - 不会上传到任何服务器
   - 卸载扩展即清除所有数据

2. **API密钥**
   - AI和地图API密钥请妥善保管
   - 不要分享给他人
   - 定期更换密钥

3. **个人信息**
   - 填写的个人资料用于生成打招呼语
   - 不会被收集或分享

### ⚠️ 法律合规

1. **用户协议**
   - 请遵守Boss直聘的用户协议
   - 不要恶意刷屏或骚扰HR

2. **使用目的**
   - 本工具仅供学习研究使用
   - 辅助求职，而非替代求职

3. **责任声明**
   - 因违规使用导致的账号封禁等后果自负
   - 作者不对使用后果承担责任

### 💡 使用建议

1. **合理设置阈值**
   - 不要设置过高的匹配阈值
   - 60-70分是比较合理的范围

2. **定期查看日志**
   - 每周查看一次日志
   - 及时发现和解决问题

3. **个性化开场白**
   - 建议填写自定义开场白
   - 比自动生成更有诚意

4. **及时回复消息**
   - 打招呼后要及时查看回复
   - 自动化只是辅助，真诚沟通更重要
